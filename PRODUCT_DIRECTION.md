# BOSSPROFIT 제품·AI 설계 기준

## 1. 제품 정의

BOSSPROFIT은 단순 원가 계산기가 아니라 소규모 외식업자를 위한 **수익 예측 및 의사결정 서비스**다.

사용자가 매일 해야 할 일은 복잡한 표를 읽는 것이 아니라 다음 세 가지다.

1. 시장 식재료 가격 흐름을 확인한다.
2. 내 매장의 재료·메뉴·판매 데이터를 쉽게 입력한다.
3. AI가 만든 예측과 권장 행동을 확인한다.

## 2. 화면과 사용자 여정

### 비로그인 영역

1. 서비스 소개
2. 식재료 시장 현황
3. 주요 품목 가격 추이 및 30·60·90일 전망
4. 회원가입 또는 로그인

개인 매장 데이터는 비로그인 홈에서 보여주지 않는다.

### 회원가입

한 화면에서 모든 정보를 요구하지 않는다.

1. 아이디 입력
2. 비밀번호 입력
3. 약관 확인
4. 가입 완료

각 단계가 완료된 뒤 다음 단계를 애니메이션으로 공개한다.

### 로그인 후 최초 온보딩

1. 매장 기본정보
   - 매장명
   - 업종
   - 지역
2. 주요 재료 등록
   - KAMIS 품목 검색
   - 직접 입력
   - 최근 구매가격 입력
3. 첫 메뉴 등록
   - 메뉴명
   - 판매가
4. 레시피 구성
   - 등록한 재료를 선택
   - 메뉴 1개당 사용량 입력
5. 최근 판매 데이터
   - 날짜별 판매량 직접 입력
   - CSV 업로드
6. 첫 분석 완료

한 단계를 저장하기 전에는 다음 단계를 노출하지 않는다.

### 로그인 후 홈

홈은 다음 네 가지만 보여준다.

1. 이번 달 예상 매출·이익
2. 가장 중요한 위험 또는 기회 한 가지
3. `메뉴 추가`, `재료 가격 입력`, `판매량 입력` 빠른 행동
4. 30·60·90일 예측으로 이동하는 버튼

세부 표와 모든 KPI는 분석 화면으로 이동한다.

### 시장 식재료 화면

개인 데이터와 분리된 공개 화면이다.

- 전체 식재료 가격 지수
- 상승·하락 품목
- 품목·지역·도매/소매 필터
- 실제 KAMIS 가격 추이
- 30·60·90일 가격 전망
- 데이터 기준일과 예측 신뢰구간

### 내 매장 재료 화면

- 내 재료 목록
- 최근 구매가격
- KAMIS 시장가격과 내 구매가격 비교
- 구매가격 변경 이력
- 메뉴 원가에 미치는 영향
- 재료 추가 마법사

## 3. 데이터 모델

필수 신규 모델:

- `Store`: 매장과 사용자 소유관계
- `StoreMember`: 향후 직원 초대를 위한 권한
- `IngredientMarketItem`: KAMIS 품목 코드와 표준 단위
- `MarketPriceObservation`: 날짜·지역·도소매·품목별 시장가격
- `StoreIngredient`: 내 매장 재료와 KAMIS 품목 연결
- `PurchasePriceObservation`: 실제 구매가격 이력
- `DailyMenuSale`: 날짜별 메뉴 판매량
- `ForecastRun`: 모델 버전, 학습 구간, 예측 구간, 성능
- `ForecastPoint`: 날짜별 중앙값과 예측구간

메뉴·재료·레시피·스냅샷은 모두 `Store`에 귀속시킨다.

## 4. 사용할 예측 모델

### 메뉴 판매량 예측: LightGBM Quantile Global Model

초기 운영 모델은 **LightGBM 분위수 회귀 글로벌 모델**로 한다.

입력 특징:

- 판매량 lag 1·7·14·28일
- 7·14·28일 이동평균과 변동성
- 요일·주말·공휴일·월
- 메뉴 가격과 가격 변경
- 메뉴 카테고리
- 날씨
- 관련 식재료 KAMIS 가격 지수
- 배달·홀·포장 채널

선택 이유:

- 한 매장의 30일 데이터만 따로 학습하지 않고 여러 메뉴와 매장의 시계열을 함께 학습할 수 있다.
- 적은 로컬 데이터에서도 범주형·달력·가격·날씨 특성을 함께 사용할 수 있다.
- 비선형 관계를 처리하면서 특징 중요도와 SHAP 설명을 제공할 수 있다.
- 분위수 손실로 중앙 예측뿐 아니라 80%·95% 예측구간을 직접 생성할 수 있다.

30일 데이터 정책:

- 플랫폼 전체 학습 데이터가 충분하면 신규 매장의 30일 데이터로 60·90일 예측을 개인화한다.
- 플랫폼 공통 데이터가 없는 초기 단계에는 30일만으로 90일을 확정 예측하지 않는다.
- 이 경우 `낮은 신뢰도 시나리오`로 표시하고 예측구간을 넓게 제공한다.

### 식재료 가격 예측: SARIMAX + LightGBM Quantile Ensemble

KAMIS 가격은 두 모델을 결합한다.

1. `SARIMAX`
   - 추세, 자기상관, 요일·월별 계절성과 외생변수를 설명한다.
   - 통계 계수와 신뢰구간을 확인할 수 있다.
2. `LightGBM Quantile`
   - 급격한 비선형 변화, 날씨, 지역, 품목 간 상호작용을 처리한다.

롤링 백테스트에서 품목별 성능이 좋은 모델에 더 높은 가중치를 주고 앙상블한다.

연간 계절성을 학습하려면 1년보다 긴 이력이 필요하다. KAMIS 일별 API의 기본 제공 범위를 넘어서는 데이터는 월별·연도별 API를 병행하거나 aT 관련 부서에 장기 이력을 요청한다.

### 장기 확장 모델: Temporal Fusion Transformer

다음 조건을 만족한 뒤에만 TFT를 후보로 추가한다.

- 최소 수백 개 이상의 관련 시계열
- 메뉴별 6개월 이상 데이터
- 날씨·가격·프로모션 등 외생변수의 안정적 수집

TFT는 다중 시점 예측과 변수 중요도 해석에 적합하지만 현재 데이터 규모에서는 과적합 위험이 있어 1차 모델로 사용하지 않는다.

### 자연어 설명 모델: OpenAI GPT-5.4 mini

사용자 화면의 설명 모델은 **GPT-5.4 mini**를 사용한다.

역할:

- 예측 결과를 사장님 언어로 설명
- 위험 원인과 권장 행동 요약
- 가격 변경 시뮬레이션 결과 설명
- 입력 과정에서 재료명과 단위 정규화 보조

선택 이유:

- 사용자 요청마다 빠르게 반응해야 하므로 플래그십 모델보다 지연시간과 비용이 중요하다.
- 구조화 출력과 도구 호출로 계산 API의 결과만 설명하게 제한할 수 있다.
- 숫자 계산과 시계열 예측은 절대 LLM에 맡기지 않는다.

복잡한 월간 보고서 생성은 필요할 때 GPT-5.5를 비동기 작업으로 사용할 수 있다.

## 5. 통계적 검증 기준

모델은 단순히 예측값을 생성했다는 이유로 출시하지 않는다.

### 비교 기준

- 전일 값 유지
- 7일 계절 나이브
- 이동평균
- ETS

### 평가 방식

- 시간 순서를 유지한 rolling-origin backtest
- 7·30·60·90일 구간별 별도 평가
- 메뉴·품목별 평가와 전체 가중 평가를 분리

### 지표

- WAPE
- MASE 또는 RMSSE
- Pinball loss
- 80%·95% 예측구간 coverage
- 예측구간 평균 폭

### 유의성

- 동일 예측 시점의 오차 차이에 대해 Diebold-Mariano test
- 전체 메뉴·품목 집계 성능은 paired bootstrap 95% 신뢰구간
- 다수 품목 검정에는 Holm 보정
- 기준 모델 대비 개선의 95% 신뢰구간이 0을 넘을 때만 `개선됨`으로 표시

30일 데이터뿐인 신규 매장은 검정 표본이 부족하므로 개인 단위의 유의성을 주장하지 않는다. 대신 플랫폼 전체 백테스트 성능과 해당 예측의 신뢰등급을 표시한다.

## 6. AI 안전 원칙

- LLM은 원본 숫자를 생성하지 않는다.
- 모든 금액과 예측은 계산·예측 API가 제공한다.
- 설명 응답에는 모델 버전과 데이터 기준일을 포함한다.
- 신뢰도가 낮으면 권고 대신 추가 데이터 입력을 요청한다.
- 예측과 실제값을 계속 저장해 모델 드리프트를 감시한다.

## 7. 구현 순서

1. 회원가입·로그인 화면 재설계
2. `Store` 중심 멀티테넌트 데이터 구조
3. 단계형 최초 온보딩
4. 재료 등록 마법사
5. 메뉴·레시피 등록 마법사
6. 단순화된 로그인 홈
7. KAMIS 시장가격 공개 화면
8. 판매·구매가격 이력 입력
9. 예측 파이프라인과 백테스트
10. GPT 설명 계층
