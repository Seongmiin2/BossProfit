기준 문서:

## 1. 서비스 정의

BOSSPROFIT은 일반 ERP나 단순 가격 조회 서비스가 아니다.

```text
시장 변화 감지
→ 내 가게 영향 계산
→ 대응안 비교
→ 실행계획 저장
→ 실제 결과 회고
```

고객이 받는 서비스는 세 가지다.

### 가격 전망

- KAMIS 실제 가격
- 7일·30일 가격 전망
- 중앙 예측과 80% 예측구간
- 기상·수급 위험요인
- 데이터 기준일과 신뢰등급

### 내 가게 영향

- 시장가격과 실제 구매가격 비교
- 재료 가격 변화에 영향을 받는 메뉴 순위
- 메뉴별 원가 상승률
- 월 예상 이익 변화 범위
- 구매가격·사용량·판매가격 시나리오 비교

### 자영업 119

- 현재 상황과 확인된 근거
- 내 매장의 노출 재료와 메뉴
- 오늘 할 일과 1주 안에 할 일
- 가격 인상 외 대안
- 성공 기준과 중단 기준
- 행동계획 저장과 결과 회고

## 2. 브랜드 메시지

### 메인 문장

> 오르기 전에, 움직이세요.

### 서비스 키워드

```text
시장 가격 예측 · 내 가게 영향 계산 · 행동 전략 제안
```

### 한 문장 설명

> 시장 가격을 미리 보고, 내 메뉴의 손익을 계산하고, 지금 할 일을 결정합니다.

### 메시지 원칙

- 한 화면의 큰 문장은 최대 두 줄
- 카드 제목은 가능하면 12자 이내
- 설명은 최대 두 줄
- 전문 용어보다 사장님이 실제 사용하는 말
- 숫자보다 먼저 “그래서 무엇을 해야 하는지” 제시
- 측정되지 않은 정확도와 수익 개선 효과를 광고하지 않음

## 3. 시각 방향

### 핵심 이미지

`정돈된 경영 도구 + 살아 움직이는 시장 신호`

### 컬러

- 바탕: 따뜻한 백색과 옅은 회색
- 핵심 행동: 테라코타 또는 번트 오렌지
- 본문: 차콜
- 안정: 낮은 채도의 세이지 그린
- 경고: 앰버
- 위험: 제한적으로 사용하는 레드

화면 전체에 강한 색을 칠하지 않는다. 색은 상태와 행동을 구분하는 데만 사용한다.

### 형태

- 카드 모서리 8~16px
- 중첩 카드 최소화
- 아이콘은 하나의 선형 시스템
- 큰 그림자보다 얇은 테두리와 높이 차이
- 숫자는 tabular numeral
- 하나의 화면에서 강조 버튼은 한 개

## 4. 모션 원칙

애니메이션은 장식이 아니라 다음 상태를 이해시키는 역할을 맡는다.

### 사용

- 화면 진입: 키워드와 카드가 60~80ms 간격으로 순차 등장
- 단계 완료: 완료 카드가 축소되고 다음 단계가 열림
- 숫자 변화: 이전 값에서 새 값으로 짧게 전환
- 그래프: 실제값 이후 예측선과 구간이 순서대로 나타남
- 저장 성공: 버튼이 체크 상태로 변한 뒤 다음 행동 표시
- 119 답변: `상황 → 영향 → 행동` 블록이 순차 공개
- 하단 시트: 모바일 입력과 상세 근거

### 금지

- 계속 반복되는 배경 움직임
- 의미 없는 아이콘 회전과 튕김
- 숫자를 오래 세는 애니메이션
- 모든 카드에 hover 이동
- 정보 확인을 늦추는 긴 전환

### 시간

- 버튼·선택: 160~180ms
- 카드·시트: 220~280ms
- 단계 전환: 280~360ms
- `prefers-reduced-motion`에서는 이동 제거

## 5. 전체 정보구조

모바일 하단 메뉴는 네 개로 고정한다.

```text
오늘 | 입력 | 시장 | 119
```

### 오늘

내 가게가 지금 해야 할 가장 중요한 행동을 보여준다.

### 입력

재료·구매가격·메뉴·판매량을 빠르게 기록한다.

### 시장

시장 브리핑과 전체 식재료 가격 현황, 품목 전망을 조회한다.

### 119

시장 상황이 내 가게에 미칠 영향을 질문하고 대응계획을 만든다.

계정·매장·상세 ERP 관리는 우측 상단 `내 정보`에서 진입한다.

## 6. UX 핵심 흐름

### 흐름 A. 처음 시작

```text
공개 홈
→ 회원가입
→ 매장 등록
→ 첫 재료
→ 구매가격
→ 첫 메뉴·레시피
→ 판매량
→ 첫 분석
```

원칙:

- 현재 단계만 확장
- 다음 단계는 제목만 미리 표시
- 저장 성공 후에만 다음 단계 공개
- 완료 단계는 한 줄 요약으로 축소
- 중단 후 재로그인하면 마지막 단계에서 재개

### 흐름 B. 재료 등록

```text
재료명 입력
→ KAMIS 후보 최대 3개
→ 사용자 선택 또는 연결 보류
→ 구매 수량·단위·가격 입력
→ 표준단위 가격 확인
→ 시장가격과 비교
→ 메뉴에 연결
```

한 화면에 모든 입력란을 노출하지 않는다.

### 흐름 C. 오늘

데이터가 부족한 경우:

```text
“판매량 3일치를 더 입력하세요”
→ 바로 입력
```

데이터가 충분한 경우:

```text
가장 중요한 알림 1개
→ 월 예상 이익
→ 빠른 입력
→ 시장 전망 또는 119
```

첫 화면의 주요 카드는 최대 3개다.

### 흐름 D. 시장

```text
오늘의 시장 브리핑
→ 거래량 TOP3
→ 오늘 변동 TOP3
→ 내일 예상 변동 TOP3
→ 품목 검색
→ 실제 가격
→ 7일·30일 전망
→ 예측구간·신뢰도
→ 기상·수급 근거
→ 내 가게 영향 보기
```

시장 첫 화면에는 개인 매장 데이터를 바로 섞지 않는다. 품목 상세에서 사용자가 `내 가게 영향`을 선택했을 때 연결한다.

#### 시장 브리핑 순위 정의

| 보드 | 계산 기준 | 표시 조건 |
|---|---|---|
| 거래량 TOP3 | 최신 완료 거래일의 표준화 가능한 도매시장 거래량·반입량 합계 | 단위를 비교할 수 있는 품목만 포함 |
| 오늘 변동 TOP3 | 오늘 가격과 직전 유효 거래일 가격의 절대 등락률 | 두 날짜 모두 유효 관측값이 존재 |
| 오늘 급등 TOP3 | 전 거래일 대비 상승률 내림차순 | 가격 단위·품종·등급이 동일 |
| 오늘 급락 TOP3 | 전 거래일 대비 하락률 오름차순 | 가격 단위·품종·등급이 동일 |
| 내일 예상 변동 TOP3 | 오늘 가격 대비 내일 중앙 예측값의 절대 예상 등락률 | 운영 채택 모델이며 신뢰등급 기준 이상 |

모든 보드에는 다음을 표시한다.

- 기준일과 마지막 수집 시각
- 품목·품종·등급·지역
- 비교 단위
- 현재 가격과 등락률
- 예측 보드의 80% 예측구간과 신뢰등급
- 데이터 출처
- 데이터 부족 또는 stale 상태

`소비량 TOP3`는 공식 소비량 통계가 확보되기 전까지 제공하지 않는다. 도매시장 거래량을 소비량이라고 표현하지 않는다.

### 흐름 E. 내 가게 영향

```text
영향 메뉴 순위
→ 가장 위험한 메뉴 선택
→ 원가·이익 변화 범위
→ 시나리오 조정
→ 119에 대응안 묻기
```

### 흐름 F. 자영업 119

```text
추천 질문 또는 직접 질문
→ 지금 상황
→ 내 매장 영향
→ 오늘 할 일
→ 대안 비교
→ 성공·중단 기준
→ 실행계획 확인
→ 사용자 승인 후 저장
```

숫자는 계산 API 결과와 완전히 일치해야 하며, 답변 하단에서 기준일·출처·모델 버전을 펼쳐볼 수 있어야 한다.

## 7. 공개 홈 구성

### 첫 화면

```text
식재료 가격 대응 플랫폼

오르기 전에,
움직이세요.

시장 가격 예측 · 내 가게 영향 계산 · 행동 전략 제안

[내 매장 시작하기]
```

오른쪽에는 세 단계만 보여준다.

```text
시장 가격 예측
→ 내 가게 영향 계산
→ 행동 전략 제안
```

### 두 번째 영역

서비스 세 개:

1. 시장 가격 예측
2. 내 가게 영향 계산
3. 행동 전략 제안

### 세 번째 영역

짧은 실제 이용 흐름:

```text
양파 등록
→ 시장 연결
→ 어묵우동 영향 확인
→ 대응계획 저장
```

### 네 번째 영역

데이터 원칙:

- 매장별 데이터 분리
- 예측 범위와 신뢰도 공개
- AI 답변 근거 표시

## 8. 화면별 핵심 카피

| 화면 | 핵심 문장 | 주 행동 |
|---|---|---|
| 공개 홈 | 오르기 전에, 움직이세요. | 내 매장 시작하기 |
| 오늘 | 오늘은 이것부터 보세요. | 우선 행동 실행 |
| 입력 | 하나씩 입력하면 분석이 완성돼요. | 현재 단계 저장 |
| 시장 | 지금 오르는 재료를 확인하세요. | 품목 보기 |
| 품목 상세 | 가격의 방향과 범위를 확인하세요. | 내 가게 영향 |
| 내 가게 영향 | 가장 먼저 영향을 받는 메뉴입니다. | 대응안 비교 |
| 119 | 상황을 알려주면 선택지를 계산할게요. | 질문하기 |
| 행동계획 | 실행할 한 가지를 정하세요. | 확인 후 저장 |
| 내 정보 | 가게와 계정을 관리하세요. | 변경 저장 |

## 9. 검증 시나리오

5명의 음식점 운영 경험자에게 설명 없이 다음 과업을 요청한다.

1. 서비스가 무엇을 해주는지 10초 안에 설명
2. 회원가입 후 첫 분석까지 완료
3. 재료 구매가격 기록
4. 양파의 30일 전망 확인
5. 양파 상승 시 영향 메뉴 찾기
6. 가격 인상 외 대응안 저장

측정:

- 10초 서비스 이해율
- 과업 성공률
- 화면별 체류시간
- 잘못 누른 횟수
- 도움 요청 지점
- 가입 후 첫 분석까지 걸린 시간

합격 기준:

- 5명 중 4명 이상이 핵심 과업 완료
- 서비스 정의를 `가격 예측 + 내 가게 영향 + 대응 전략` 중 두 가지 이상으로 설명
- 모바일 360px에서 가로 스크롤 없음
- 입력 오류 후 기존 값 유지
- 사용자가 다음 행동을 묻지 않아도 스스로 진행

## 10. 구현 순서

검증 후 아래 순서로 구현한다.

1. 공개 홈 카피와 세 서비스 구조
2. 앱 셸을 `오늘 | 입력 | 시장 | 119`로 변경
3. 단계형 재료·구매가격 입력
4. 오늘 화면 우선순위 재설계
5. 시장 현황과 품목 상세
6. 내 가게 영향
7. 119와 행동계획
8. 모션·접근성·반응형 최종 조정

기능 데이터가 없는 화면에는 실제값처럼 보이는 가짜 숫자를 사용하지 않는다. UI 데모 수치는 반드시 `예시`로 표시한다.
