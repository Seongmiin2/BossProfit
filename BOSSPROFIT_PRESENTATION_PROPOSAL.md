# BOSSPROFIT 발표용 기획서

> 가격이 오른 뒤 계산하지 않고, 오르기 전에 준비하는 식당

발표 목적: 문제의 시급성, 데이터 적합성, 모델 설계의 필연성, 성능 검증 방법, 사용자 가치를 하나의 이야기로 설득한다.

---

## 1. 핵심 메시지

BOSSPROFIT은 식재료 가격을 단순히 보여주는 ERP가 아니다.

시장 가격, 주산지 기상, 도매시장 거래량과 사용자의 실제 매장 데이터를 연결해:

1. 앞으로의 가격 범위를 예측하고,
2. 어떤 메뉴의 원가와 이익이 영향을 받는지 계산하며,
3. 사장님이 지금 실행할 대응 전략을 제안하고,
4. 실행 결과까지 기록하는 AI 경영 안전망이다.

---

## 2. 문제의식

### 외식업의 가장 큰 변동비가 가장 예측하기 어려운 비용이다

- 외식업체의 94.1%가 식재료비 상승을 경영상 어려움으로 응답했다.
- 영업비용 중 식재료비는 42.4%로 가장 큰 비중을 차지한다.
- 한국은행 분석에 따르면 월평균기온이 1℃ 상승할 때 농산물가격 상승률은 최대 0.44%p 높아진다.

자영업자는 식재료 가격에 직접 노출되지만 납품 가격이 오른 뒤에야 변화를 안다. 메뉴 가격은 고객 저항 때문에 식재료 가격만큼 빠르게 바꿀 수 없다.

따라서 손실은 가격 상승 순간보다 대응이 늦어진 기간 동안 누적된다.

### 이상기후는 작은 생산 감소를 큰 가격 충격으로 증폭한다

- 2022년 고온·가뭄: 양파 생산량 8% 감소, 가격 89.8% 상승
- 2023년 저온·집중호우: 사과 생산량 23% 감소, 가격 91.4% 상승
- 2024년 폭염·가뭄: 여름배추 생산량 9% 감소, 도매가격 49.3% 상승

농산물은 생산량을 즉시 늘릴 수 없으므로 기상 충격과 가격 변화가 선형적으로 움직이지 않는다. 가격 이력만 학습하는 모델로는 기상 충격이 가격에 반영되기 전에 대응하기 어렵다.

## 3. 기존 방식의 한계

### 일반 ERP

- 사용자가 입력한 과거 데이터를 저장한다.
- 매출, 원가, 이익을 표로 보여준다.
- 시장 변화의 의미와 다음 행동은 사용자가 해석해야 한다.

### 단순 시계열 예측

- 어제 가격으로 내일 가격을 예측한다.
- 계절성과 추세는 학습하지만 갑작스러운 폭염·호우·한파를 늦게 반영한다.
- 특정 품목과 시기에 반복되는 과대·과소 예측 편향을 그대로 남긴다.

### 일반 RAG 챗봇

- 시장 기사와 운영 가이드는 검색할 수 있다.
- 정확한 원가, 미래 가격, 메뉴별 손실을 직접 계산하기 어렵다.
- 내 매장의 레시피와 판매량에 연결되지 않으면 일반적인 조언에 그친다.

## 4. 제안 서비스

### 평상시: 가격 전망

- 1일·7일·30일 식재료 가격 전망
- 중앙값과 80% 예측구간
- 기상·수급·잔차 보정량 공개
- 시장 평균과 내 구매가격 비교

### 매장 분석: 내 가게 영향

- 재료 가격 변화에 따른 메뉴별 원가 상승률
- 월 예상 이익 변화
- 영향을 많이 받는 메뉴 순위
- 가격·사용량·판매량 변경 시나리오

### 위기 대응: 자영업 119

- 현재 발생한 사건과 근거
- 내 매장에 미치는 영향
- 예상 손실 범위
- 오늘과 1주 안에 할 일
- 가격 인상 외 대안
- 성공 기준과 중단 기준
- 행동계획 저장과 결과 회고

## 5. 데이터 구성

### 가격

- KAMIS 1996년 이후 일별 도소매 가격
- 품목·품종·등급·지역·시장 구분

### 수급

- 전국 공영도매시장 경락가격
- 거래량과 반입량
- 산지와 시장별 수급 흐름

### 기상

- 기상청 ASOS 관측자료
- 농촌진흥청 농업기상과 토양수분
- 기상청 단기·중기·1개월 전망
- 폭염·호우·한파·태풍 특보

### 생산

- 품목별 주산지
- 재배면적, 단수, 생산량
- 생육단계와 작부체계
- 병해충 발생정보

### 매장

- 실제 구매가격
- 메뉴와 레시피
- 판매량
- 채널 수수료와 포장비

모든 데이터는 단순히 날짜로 결합하지 않는다. 해당 예측 시점에 실제로 공개되어 있던 정보만 결합한다.

## 6. 모델 구조

### Stage 1. Base Price Model

가격, 계절성, 시장 거래량의 기본 흐름을 학습한다.

후보:

- Seasonal Naive
- SARIMAX
- LightGBM Quantile
- 검증 성능 기반 앙상블

### Stage 2. Weather & Supply Impact Model

기상 데이터를 가격에 바로 입력하지 않는다.

```text
주산지 기상
→ 생육·수확 충격
→ 출하량·거래량 변화
→ 가격 보정량
```

품목별 주산지와 생육단계를 반영해 같은 강수와 기온이라도 시기와 지역에 따라 다른 영향을 학습한다.

### Stage 3. Residual Correction Model

1·2단계 예측과 실제 가격의 차이를 학습한다.

```text
residual(t, h)
= log(actual_price(t+h))
- log(stage2_prediction(t, h))
```

단, 학습 데이터에 다시 맞춘 예측값은 사용하지 않는다. Rolling-origin 방식으로 생성한 OOF 예측만 잔차 label로 사용한다.

### 최종 예측

```text
최종가격
= 기본가격
+ 기상·수급 보정량
+ 잔차 보정량
```

## 7. 성능을 증명하는 실험

### 비교 모델

1. Last Value
2. Seasonal Naive
3. Base Price Model
4. Base + Weather & Supply
5. Base + Weather & Supply + Residual

### 검증 방식

- Expanding-window rolling-origin evaluation
- 1일·7일·30일 horizon 개별 평가
- 평시와 기상 충격 기간 분리
- 품목별 독립 결과
- Untouched test set 유지

### 지표

- WAPE
- MASE
- MAE·RMSE
- Bias
- Pinball Loss
- 80% Prediction Interval Coverage
- Interval Width

### 출시 Gate

아래 값은 측정 결과가 아니라 모델을 운영에 채택하기 위한 사전 기준이다.

| 검증 영역 | 출시 기준 |
|---|---:|
| 전체 오차 | Seasonal Naive 대비 WAPE·MASE 10% 이상 개선 |
| 기상 충격 | Base Model 대비 충격 기간 MAE 15% 이상 개선 |
| 잔차 보정 | Weather Model 대비 untouched test 3% 이상 추가 개선 |
| 예측구간 | 80% 구간 coverage 76~84% |
| 편향 | 품목·horizon별 평균 bias ±3% 이내 |

보정 단계가 특정 품목과 horizon의 test 성능을 떨어뜨리면 해당 보정량은 0으로 비활성화한다.

## 8. 데이터 누수 방지

- 무작위 train/test 분할 금지
- 미래 실제 가격 사용 금지
- 미래 실제 관측 기상을 예보처럼 사용 금지
- 예보의 issued_at과 valid_at 저장
- 전처리와 scaling은 학습 구간에서만 fit
- 잔차 label은 OOF 예측으로만 생성
- 실제 미래 기상을 사용한 실험은 Oracle Experiment로 별도 표기

이 규칙을 지키지 않은 높은 정확도는 서비스 성능으로 인정하지 않는다.

## 9. 사용자에게 전달되는 결과

예시 질문:

> 다음 달 양파값이 더 오르면 가격 인상 말고 무엇을 할 수 있어?

예시 답변:

- 양파의 30일 가격은 현재보다 7~13% 높아질 가능성이 있다.
- 가장 영향을 많이 받는 메뉴는 어묵우동이다.
- 현재 판매량 기준 월 이익이 약 8만 4천 원 감소할 수 있다.
- 우선 2주 고정가 공급계약을 문의한다.
- 양파 사용량을 5% 조정하면 예상 손실 일부를 줄일 수 있다.
- 2주 후 주문량과 실제 원가를 점검한다.

예시 수치는 기능 설명용이며 실제 모델 측정 결과가 아니다.

## 10. 차별점

| 일반 ERP | BOSSPROFIT |
|---|---|
| 과거 데이터 저장 | 시장·기상·수급·매장 데이터 연결 |
| 원가와 매출 표 | 미래 가격과 메뉴별 영향 |
| 사용자가 해석 | AI가 근거와 대안 제시 |
| 전략 실행과 단절 | 행동계획과 실제 결과 추적 |

핵심 경쟁력은 챗봇이 아니라:

```text
예측
→ 내 가게 영향 계산
→ 전략 선택
→ 실행
→ 결과 회고
→ 다음 전략 개선
```

으로 이어지는 폐쇄형 데이터 학습 루프다.

## 12. 결론

가격 변동 자체를 막을 수는 없다.

그러나 가격이 오르기 전에 영향을 계산하고, 가격 인상 외의 대안을 준비하도록 만들 수는 있다.

BOSSPROFIT은 시장 정보를 보여주는 서비스가 아니라:

> 식재료 가격 변동을 예측하고, 내 메뉴의 손익에 연결하며, 사장님과 실행 전략을 세우는 AI 자영업 경영 안전망이다.
